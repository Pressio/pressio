Public namespace\+: {\ttfamily pressio\+::linearsolvers}\hypertarget{md_pages_components_linsolvers_autotoc_md2}{}\doxysection{Overview}\label{md_pages_components_linsolvers_autotoc_md2}
First, we want to clearly state that this subpackage of pressio does not provide linear solvers classes implemented from scratch that work in general. This is an intentional choice\+: there exist already many open-\/source highly-\/optimized libraries that do so in both a shared-\/memory, e.\+g. Armadillo, Eigen, Blaze, Magma, LAPACK, and distributed environment, e.\+g. Trilinos, PETSc, HYPRE.

The main goal of the pressio\textquotesingle{}s linear solvers component is to support usecases that are needed for ROMs. As discussed in the introduction, ROMs involve \char`\"{}small\char`\"{} {\itshape dense} systems that suitably fit on a {\itshape single compute node}. Therefore, pressio\textquotesingle{}s linear solvers are designed and implemented to be wrappers (exposing a common interface) to existing shared-\/memory linear solvers libraries.

Currently, we support solving linear systems that involve either Eigen or Kokkos data structures. The reason for this is that Eigen and Kokkos are the preferred choices (for now) to implement ROMs operators. Kokkos, in particular, allows to operate both on host and GPUs. Later on, we can easily extend support to other libraries if needed.\hypertarget{md_pages_components_linsolvers_autotoc_md3}{}\doxysection{API}\label{md_pages_components_linsolvers_autotoc_md3}
