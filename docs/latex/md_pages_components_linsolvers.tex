Public namespace\+: {\ttfamily pressio\+::linearsolvers} \hypertarget{md_pages_components_linsolvers_autotoc_md3}{}\doxysection{Overview}\label{md_pages_components_linsolvers_autotoc_md3}
We clearly state upfront that this component of pressio is not a self-\/contained, general purpose library of linear solvers implemented from scratch. In the future, we might revise this, but for now it is an {\bfseries{intentional}} choice\+: there exist already many open-\/source highly-\/optimized libraries that provide linear solvers for both shared-\/memory, e.\+g. Armadillo, Eigen, Blaze, Magma, LAPACK, and distributed environment, e.\+g. Trilinos, PETSc, HYPRE.



The primary goal of the pressio\textquotesingle{}s linear solvers is to support usecases that are needed for ROMs. As discussed in the introduction, ROMs involve \char`\"{}small\char`\"{} {\itshape dense} systems that suitably fit on a {\itshape single compute node}. Pressio\textquotesingle{}s linear solvers are thus designed and implemented to be wrappers (exposing a common interface) to existing shared-\/memory linear solvers libraries. 

Currently, we support linear systems using either Eigen or Kokkos containers. The reason for this is that Eigen and Kokkos are the supported and preferred choices (for now) to implement ROMs operators. Kokkos, in particular, allows to operate both on host and GPUs. Later on, we can easily extend support to other libraries if needed.
