In order to use the R\+OM functionalities in pressio, one needs to create a F\+OM adapter class. Pressio supports two main types of A\+PI as described below.\hypertarget{md_pages_getstarted_adapter_api_autotoc_md17}{}\doxysection{Continuous-\/time A\+PI}\label{md_pages_getstarted_adapter_api_autotoc_md17}
Recall that pressio targets a generic full-\/order model (F\+OM) system written as \[ \frac{d \boldsymbol{y}}{dt} = \boldsymbol{f}(\boldsymbol{y},t; \boldsymbol{\mu}), \quad \boldsymbol{y}(0;\boldsymbol{\mu}) = \boldsymbol{y}(\boldsymbol{\mu}), \] where $y$ is the F\+OM state and $f(...)$ is the F\+OM velocity.

We envision two scenarios\+:
\begin{DoxyItemize}
\item A\+: you are only able to expose the right-\/hand-\/side (or velocity) of your F\+OM application
\item B\+: you expose the right-\/hand-\/side of your F\+OM application as well as the action of the velocity\textquotesingle{}s Jacobian on some operand
\end{DoxyItemize}\hypertarget{md_pages_getstarted_adapter_api_autotoc_md18}{}\doxysubsection{A\+: velocity only}\label{md_pages_getstarted_adapter_api_autotoc_md18}
