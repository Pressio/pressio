



A key assumption of projection-\/based ROMs is to approximate the full-\/order model (FOM) state, $y_{fom}$, as\+: \[ y_{fom} = g(y_{rom}) \]

where $y_{rom}$ is the reduced state (or generalized coordinates), and $g$ is the decoder (or mapping).

\hypertarget{md_pages_components_rom_decoder_autotoc_md71}{}\doxysection{Special case\+: linear decoder}\label{md_pages_components_rom_decoder_autotoc_md71}


Defined in header\+: {\ttfamily $<$pressio/rom\+\_\+decoder.\+hpp$>$}

Public namespace\+: {\ttfamily pressio\+::rom} 

A linear decoder is a mapping of the form\+: \[ y_{fom} = \phi y_{rom} \]

where $\phi$ is the Jacobian matrix (for the time being, assume it constant).

Pressio offers this functionality via the following function\+:


\begin{DoxyCode}{0}

\end{DoxyCode}



\begin{DoxyItemize}
\item {\ttfamily Fom\+State\+Type}\+: type of your FOM state, must be copy constructible
\item {\ttfamily Jacobian\+Type}\+: type of the decoder\textquotesingle{}s Jacobian, must be copy constructible and move constructible
\end{DoxyItemize}

Obviously, an instance of a linear decoder object meets the decoder concept discussed above.
