Public namespaces\+: {\ttfamily pressio} and {\ttfamily pressio\+::utils}.\hypertarget{md_pages_components_utils_autotoc_md85}{}\doxysection{Logger}\label{md_pages_components_utils_autotoc_md85}
One of the main functionalities inside {\ttfamily utils} is the logger. To implement the pressio logging functionalities, we have used \href{https://github.com/gabime/spdlog}{\texttt{ spdlog}}, and rewritten a subset of it, for example to work seamlessly with MPI.



\begin{DoxyParagraph}{By default all logging is disabled.}
If you just {\ttfamily \#include$<$pressio/utils.\+hpp$>$} and expect logging, you will be disappointed! By default, all logging is disabled for performance reasons. To enable and use it, you need to do two things\+:
\begin{DoxyEnumerate}
\item place a {\ttfamily define} statement to set the desired level\+: note that this define statement {\itshape must} be placed {\itshape before} including the utils header.
\item you need to start and finalize the logger singleton.
\end{DoxyEnumerate}
\end{DoxyParagraph}
As follows\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{comment}{// this sets the default min level}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define PRESSIO\_LOG\_ACTIVE\_MIN\_LEVEL 2}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// this is needed to include the logger code}}
\DoxyCodeLine{\textcolor{preprocessor}{\#include <pressio/utils.hpp>}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keywordtype}{int} main()}
\DoxyCodeLine{\{}

\DoxyCodeLine{  plog::initialize(pressio::logto::terminal);}
\DoxyCodeLine{}
