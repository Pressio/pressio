Public namespaces\+: {\ttfamily pressio}, {\ttfamily pressio\+::utils} \hypertarget{md_pages_components_utils_autotoc_md145}{}\doxysection{Logger}\label{md_pages_components_utils_autotoc_md145}
One of the main functionalities inside {\ttfamily utils} is the logger. To implement the pressio logging functionalities, we have used \href{https://github.com/gabime/spdlog}{\texttt{ spdlog}}, modified a subset of it, for example to work seamlessly with MPI.



By default, for performance reasons, the logger is disabled, so pressio will not output anything. 



To enable logging, you need to do two things\+:
\begin{DoxyEnumerate}
\item insert a {\ttfamily define} statement to set the {\itshape minimum} level {\itshape before} including the utils header
\item initialize and finalize the logger singleton 
\end{DoxyEnumerate}

The following snippet provides the main idea\+:

\DoxyCodeLine{\textcolor{comment}{// this is needed since it contains the logging code}}

\DoxyCodeLine{}

\hypertarget{md_pages_components_utils_autotoc_md147}{}\doxysubsection{If running with MPI}\label{md_pages_components_utils_autotoc_md147}
If you are running with MPI, the logger prints to the terminal {\itshape only from rank==0}. However, it automatically creates a per-\/rank log file if you choose the file output. For example, the following code\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{int} main(\textcolor{keywordtype}{int} argc, \textcolor{keywordtype}{char} **argv)}
\DoxyCodeLine{\{}
\DoxyCodeLine{  MPI\_Init(\&argc, \&argv);}
\DoxyCodeLine{  \textcolor{keywordtype}{int} my\_rank = \{\};}
\DoxyCodeLine{  MPI\_Comm\_rank(MPI\_COMM\_WORLD, \&my\_rank);}
\DoxyCodeLine{}
\DoxyCodeLine{  \textcolor{keyword}{namespace }plog = pressio::log;}
\DoxyCodeLine{  plog::initialize(pressio::logto::file, \textcolor{stringliteral}{"{}log\_file.txt"{}});}
\DoxyCodeLine{  PRESSIOLOG\_INFO(\textcolor{stringliteral}{"{}print from rank \{:2\}"{}}, my\_rank);}
\DoxyCodeLine{  plog::finalize();}
\DoxyCodeLine{}
\DoxyCodeLine{  MPI\_Finalize();}
\DoxyCodeLine{\}}

\end{DoxyCode}


If we were to run this with N ranks, we would obtain two files {\ttfamily log\+\_\+file.\+txt\+\_\+0}, and {\ttfamily log\+\_\+file.\+txt\+\_\+1}. Currently, the logger works only for the world communicator. \hypertarget{md_pages_components_utils_autotoc_md149}{}\doxysubsection{Resetting the level}\label{md_pages_components_utils_autotoc_md149}
If you want, you can use the define statement to set the min level, but then at runtime you can reset for a {\itshape higher} level (see below). Note that you cannot reset the level to something that is {\itshape lower} than the one you set via the {\ttfamily define} statement.

\hypertarget{md_pages_components_utils_autotoc_md150}{}\doxysubsection{The loggin macros}\label{md_pages_components_utils_autotoc_md150}
To actually issue log statements, you use the macros as in the following example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{int} main()}
\DoxyCodeLine{\{}
\DoxyCodeLine{  \textcolor{comment}{// initialize logger}}
\DoxyCodeLine{}
\DoxyCodeLine{  \textcolor{keywordtype}{double} a = 1.1;}
\DoxyCodeLine{  PRESSIOLOG\_TRACE(\textcolor{stringliteral}{"{}my value is \{:.6f\}"{}}, a);}
\DoxyCodeLine{  PRESSIOLOG\_DEBUG(\textcolor{stringliteral}{"{}my value is \{:.6f\}"{}}, a);}
\DoxyCodeLine{  PRESSIOLOG\_INFO(\textcolor{stringliteral}{"{}my value is \{:.6f\}"{}}, a);}
\DoxyCodeLine{  PRESSIOLOG\_WARN(\textcolor{stringliteral}{"{}my value is \{:.6f\}"{}}, a);}
\DoxyCodeLine{  PRESSIOLOG\_ERROR(\textcolor{stringliteral}{"{}my value is \{:.6f\}"{}}, a);}
\DoxyCodeLine{  PRESSIOLOG\_CRITICAL(\textcolor{stringliteral}{"{}my value is \{:.6f\}"{}}, 55.6);}
\DoxyCodeLine{}
\DoxyCodeLine{  \textcolor{comment}{// finalize logger}}
\DoxyCodeLine{\}}

\end{DoxyCode}


where we note that you can use the \href{https://github.com/fmtlib/fmt}{\texttt{ \{fmt\} library}} to properly format the print statements.



\begin{DoxyParagraph}{Keep in mind\+:}
The log statements issued for a specific level will be printed {\itshape only if} {\ttfamily PRESSIO\+\_\+\+LOG\+\_\+\+ACTIVE\+\_\+\+MIN\+\_\+\+LEVEL} is smaller or equal than that level. If the logger is disabled, the macros are expanded to a no-\/op. So it does not cost you anything to place log statements in your code, because in production mode you can just compile to no-\/op. 
\end{DoxyParagraph}
